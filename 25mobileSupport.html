    Mobile Responsiveness

    Our website is not mobile friendly. We will modify the CSS to make it responsive. We will use media-query. We will convert our app.js to styled components. We create a new file global.styles.js where we imort { createGlobalStyle } from 'styled-components'; Then we const GlobalStyle = createGlobalStyle` ... ` where we copy the CSS from our App.css file. Then we export these styles out. In our App.tsx we import { GlobalStyle } from "./global.styles";. We then place a self closing GlobalStyle Component inside the div over the Header Component. We use @media screen and (max-width: 800px) { ... } will apply the style we set inside the brackets when the screen doesn't exceed 800px. We set padding: 10px; which will change the padding in body to 10px insetad of what we have set above, when the screen is under 800px.

    Header Component
    We start with Header. We use the same media query, which we will be copying all around. We start with making the header 10px smaller in height, with a padding of 10px, and a small margin of 20px in the bottom part. Then we change the logo to have a width of 50px and a padding of 0. Then we make the options container to take up 85% of the width, which is enough to display the options properly.

    MenuItem Component
    This one was slightly more complicated. I had decided to handle the dynamic size of the items in the home page using inline styles because of being unable of passing the Props to MenuItemContainer with TS. I found that you can do this by adding a generic in the styles file styled.div<GENERIC> which is the prop that we will be passing. This allows us to use Props in our styles and not have TS err. After doing that I was setting the height dynamically inside of styled components which allowed me to set the media query with height 200px in the element. This works because media queries have heigher specificity than any regular selector. Doing this in the component would have required a complex hook.

    ShopPage: CollectionPreview, CollectionItem
    We started with CollectionItem which I had to refactor to use the variables inside styled components as well. Once that was set up we use the media query to set the width to 40vw, and we want to remove the hover features, because we don't use them in mobile. We copy the hover code into the media query and set opacity: unset; on both selectors.
